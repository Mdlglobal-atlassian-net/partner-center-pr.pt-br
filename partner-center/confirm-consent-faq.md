## <a href="" id="processfaq"></a>Processo

**O que é o novo processo para confirmar se a aceitação do cliente da Microsoft Cloud MCA (contrato)?**

A Microsoft está comprometida em ajudar nossos clientes e parceiros permanecerá em conformidade. Como um parceiro no programa de provedor de soluções de nuvem (CSP) da Microsoft, você é responsável por termos da Microsoft revisão e condições, incluindo o uso aceitável dos produtos e serviços com clientes. Para melhor auxiliar os parceiros atendendo aos requisitos de conformidade e certifique-se de transparência, estamos permitindo que um novo processo que exige que os parceiros CSP confirmar que seus clientes aceitou os termos MCA. A partir de 7 de agosto de 2018, apresentamos um novo campo no painel do Partner Center interface do usuário também uma API do Partner Center para parceiros para fornecer essas informações. Inicialmente será opcional fornecer essa confirmação, no entanto, será necessário a partir de 7 de novembro de 2018. 

**Quando esse requisito entram em vigor?**

Os parceiros podem começar confirmando aceitação assim que a funcionalidade está disponível em 7 de agosto de 2018. Os usuários do painel do Partner Center a partir de 7 de novembro de 2018, são necessárias para confirmar a aceitação de cliente do MCA em todas as transações de novo. Em 22 de março de 2019 usuários de API do Partner Center será necessários para fazer o mesmo. O prazo para usuários de API foi estendido para permitir que eles adicionais tempo para implementação técnica.

**NOVOS | Microsoft valida a precisão das informações do cliente fornecidas pelo parceiro?**

Microsoft valida se o endereço de email do cliente fornecido é de formato correto. É responsabilidade do parceiro para fornecer informações precisas e corrigir qualquer informação encontrada imprecisas. 

**NOVOS | Como pode um parceiro provar aceitação do cliente do MCA, se ele está assinado, documentado, digitalizadas, ou em papel?**

O parceiro é responsável por garantir que o cliente aceitou os termos do MCA e para gerenciar seu próprio processo e registros. Usando o processo de painel do Partner Center ou a API do Partner Center, o parceiro confirma que o cliente aceitou os termos do MCA.

**NOVOS | Como Microsoft verificar que o processo de confirmação for concluído para um cliente?**

A partir 7 de novembro de 2018, quando um parceiro CSP inicia uma transação usando o painel do Partner Center para um cliente novo ou existente, sistemas da Microsoft automaticamente verificará se as informações de confirmação do cliente estão presentes no Partner Center Painel de controle. Se não estiver presente, a transação não poderão ser concluída. 

A partir 22 de março de 2019, quando um parceiro CSP inicia uma transação para um cliente novo ou existente usando a API do Partner Center, sistemas da Microsoft automaticamente verificará se as informações de confirmação do cliente estão presentes. Se não estiver presente, a transação não poderão ser concluída. 

**NOVOS | Notificação é enviada para clientes cujo endereço de email é fornecido no processo de confirmação?**

Não há nenhum processo de notificação.

## <a href="" id="policyfaq"></a>Política

**Por que a Microsoft está exigindo parceiros confirmar que os clientes aceitou o MCA?** 

O Microsoft Cloud MCA (contrato) protege Microsoft, os clientes e parceiros, fornecendo direitos de uso do cliente, termos e condições concedidas para pedidos de produtos e serviços do Microsoft Cloud. 

Execução apropriada do MCA protege os nossos parceiros, os clientes e da Microsoft, garantindo mútuo alinhamento em muitos tópicos importantes, incluindo, mas não limitado a segurança, privacidade e proteção de dados. O processo de confirmação garante a transparência e o alinhamento e permite que a Microsoft ajudar melhor os clientes que precisam responder prontamente consultas regulatórias.

**Quais informações de cliente são necessárias para concluir o processo de confirmação?**
As informações a seguir são necessárias em relação a pessoa que aceitem o contrato:
-   Data de aceitação do contrato
-   Nome e sobrenome
-   Email
-   Número de telefone (opcional) 

**NOVOS | O que acontece se nós não atenderem a data de vencimento?**

Os parceiros podem começar confirmando aceitação assim que a funcionalidade está disponível em 7 de agosto de 2018. Os usuários do painel do Partner Center a partir de 7 de novembro de 2018, são necessárias para confirmar a aceitação do cliente do MCA em todas as transações de novos. Em 22 de março de 2019 usuários de API do Partner Center será necessários para fazer o mesmo.

No momento da imposição, novas aquisições para que um cliente existente ou alterações assento contagem nas assinaturas existentes não serão permitidas até que a aceitação do cliente seja confirmada. Para um novo cliente, transações não serão permitidas até que a aceitação for confirmada. A renovação automática não seja afetada por esse requisito.

**Quais são as regiões esse requisito se aplica a?**

Confirmação de aceitação do cliente para o MCA é necessária para todas as regiões em nuvem pública da Microsoft. Confirmação de aceitação do cliente não é compatível com nuvens nacionais (incluindo Microsoft Cloud para o governo e Microsoft Cloud Alemanha).

**Quais parceiros são responsáveis por Confirmar aceitação do cliente para o MCA?**

Efetivos 7 de novembro de 2018, parceiros fornecedores indiretos do provedor de serviços de nuvem (CSP) e cobrança direta do CSP são responsável por Confirmar aceitação do cliente e podem usar a API do Partner Center ou o painel do Partner Center para fazê-lo. Provedores indiretos talvez seja necessário primeiro coletar as informações necessárias de seus revendedores indiretos e, em seguida, disponibilizá-lo à Microsoft por meio do painel do Partner Center ou API do Partner Center. Ao contrato, ele é o provedor indireto que tem uma associação de contrato com a Microsoft para garantir que os clientes de término dos seus revendedores CSP aceitou para o MCA. 

**Quais cenários de cliente exigem a confirmação da aceitação de cliente do MCA?**

A aceitação do contrato de nuvem da Microsoft deve ser confirmada quando os parceiros CSP quer fazer as transações a seguir:
1.  Um novo cliente é concluído novas aquisições
2.  Um cliente existente for concluída novas aquisições
3.  Contagem de assentos as atualizações de um cliente existente para as assinaturas existentes

**Pode confirmar a aceitação para clientes existentes separadas do suas novas aquisições ou adições de assento?**

Sim. Aceitação do cliente poderá ser confirmada para todos os clientes cobertos pelo MCA.

**Quais são os requisitos para clientes que têm vários parceiros do CSP?**

Confirmação de aceitação será confirmada pelo parceiro. Se um cliente faz transações com vários parceiros CSP, cada parceiro precisará confirmar a aceitação.

**Sou um revendedor indireto do CSP trabalhar com provedores indiretos do CSP. O que é minha responsabilidade?**

Ao contrato, ele é o provedor indireto que tem uma associação de contrato com a Microsoft para garantir que o cliente final aceitou o MCA. No entanto, provedores indiretos podem ter disposições separadas com revendedores indiretos para coletar aceitação do cliente. Nesse caso, provedores indiretos precisará primeiro coletar as informações de aceitação do cliente necessária de seus revendedores indiretos e, em seguida, disponibilizá-lo à Microsoft por meio do painel do Partner Center ou API do Partner Center. Você pode dar suporte a esse processo, imediatamente fornecendo as informações necessárias para seus provedores indiretos após sua solicitação.

**Sou um provedor indireto do CSP. É necessário coordenar com meu revendedor indireto do CSP para estar preparado?**

Provedores indiretos que precisam para coletar as informações de aceitação do cliente necessária de seus revendedores indiretos aconselhável se comunicar esse requisito para seus revendedores indiretos imediatamente para coordenar entrega e o recebimento dessas informações.

**Como posso explicar meus clientes que eles devem confirmar sua aceitação o MCA?**

Execução apropriada do MCA protege Microsoft, seus clientes e parceiros, garantindo mútuo alinhamento em muitos tópicos importantes, incluindo, mas não limitado a segurança, privacidade e proteção de dados. O processo de confirmação garante a transparência e o alinhamento e permite que a Microsoft ajudar melhor os clientes que precisam responder prontamente consultas regulatórias.

**NOVOS | Se um cliente funciona com um único provedor indireto do CSP, mas as transações são concluídas com vários revendedores indiretos do CSP, cada revendedor indireto do CSP precisar fornecer o provedor indireto do CSP com as informações de confirmação?**

O provedor indireto do CSP é responsável por fornecer a confirmação de aceitação do cliente MCA à Microsoft. Para cada conta de cliente, apenas uma confirmação pode ser registrada no painel do Partner Center ao mesmo tempo. 

O provedor indireto do CSP é responsável por confirmar que uma pessoa de contato de cliente responsável por concordando no MCA tiver feito isso, independentemente do número de CSP revendedores indiretos que eles transações com.

**NOVOS | Quem deve ser a pessoa autorizada a confirmar a aceitação do MCA dentro da organização do cliente?**

É responsabilidade do parceiro para obter a aceitação do cliente do MCA de uma pessoa dentro da organização do cliente. É responsabilidade do cliente para determinar quem está autorizado a fornecer a aceitação. 

**NOVOS | Qual versão do MCA são responsáveis por Confirmar aceitação do cliente de parceiros? Os parceiros precisa indicar qual versão do MCA foi aceita pelo cliente?**

Os parceiros são responsáveis por usando o MCA mais atual ao fazer transações CSP com um novo cliente. Os parceiros devem confirmar a aceitação do MCA mais atual quando a renovação, alterar ou preencher novas transações com um cliente existente. 

No caso de que o MCA é atualizado antes da renovação, os parceiros não precisam ter o cliente novamente obter aceitação do cliente do MCA imediatamente, mas devem reconfirmar consentimento para o MCA antes da renovação. No entanto, você pode querer fazer com que o cliente aceitar novos termos antes da renovação para tirar proveito dos termos novos e aprimorados do MCA.

**NOVOS | Sou um revendedor indireto do CSP. Se eu gravar aceitação do cliente do MCA nos registros da minha empresa, é necessário também fornecer essas informações por meio do painel do Partner Center ou a API do Partner Center?**

Ao contrato, ele é o provedor indireto que tem uma associação de contrato com a Microsoft para garantir que o cliente final aceitou o MCA. No entanto, provedores indiretos podem ter disposições separadas com revendedores indiretos para coletar aceitação do cliente. Nesse caso, provedores indiretos precisará primeiro coletar as informações de aceitação do cliente necessária de seus revendedores indiretos e, em seguida, disponibilizá-lo à Microsoft por meio do painel do Partner Center ou API do Partner Center.

**NOVOS | E se meu cliente não concorda com os termos do MCA?**

Como um parceiro no programa CSP da Microsoft, você é responsável por aceitação do cliente do MCA antes de fazer um pedido. Isso não é um requisito de novo. Por qualquer motivo, se o cliente não concordar com o MCA, em seguida, o parceiro deve cancelar as assinaturas associadas com esse cliente.

**NOVOS | Um revendedor pode dar consentimento em nome do cliente?**

Não.

**Depois de confirmar que um cliente forneceu aceitação, é necessário confirmá-lo novamente para o cliente no futuro?**

Depois que tiver confirmado que o cliente aceitou o MCA, essas informações serão mantidas na seção da conta de cliente no Partner Center e não é necessário para torne a confirmar aceitação para a duração de tempo que o cliente está coberto pelos termos do contrato. Você só deve torne a confirmar aceitação antes de fazer um novo pedido.

No caso de que o MCA é atualizado antes da renovação, os parceiros não são necessários para que o cliente novamente obter aceitação do cliente do MCA imediatamente, mas devem torne a confirmar consentimento para o MCA antes da renovação. No entanto, você pode querer fazer com que o cliente aceitar novos termos antes da renovação para tirar proveito dos termos novos e aprimorados do MCA.

**NOVOS | Não estou confortável dando à Microsoft informações de contato direto do meu cliente final. Como a Microsoft usará essas informações?**

Quando um parceiro cria um locatário do cliente, é necessário fornecer uma informações do contato principal para o cliente. De acordo com os termos de serviços Online, "Microsoft processará somente dados pessoais instruções documentadas do cliente."

## <a href="" id="technicalfaq"></a>Técnico

**Como os parceiros confirmar a aceitação do cliente e fornecê-las ao Microsoft? Quais ferramentas estão disponíveis para dar suporte a esse processo?**

Existem dois métodos que estão disponíveis a partir de 7 de agosto de 2018 para ajudar os parceiros: 
1.  Os parceiros podem usar a API do Partner Center para confirmar programaticamente aceitação do cliente. 
2.  Os parceiros podem usar o painel do Partner Center para confirmar a aceitação do cliente.

**NOVOS | Sou um parceiro CSP que usa a API do Partner Center. Quais alterações de API é necessário considerar?** 

Nesse caso, as alterações de API para levar em conta são:

-   [https://docs.microsoft.com/en-us/partner-center/develop/get-confirmation-of-customer-consent](https://docs.microsoft.com/en-us/partner-center/develop/get-confirmation-of-customer-consent) 

-   [https://docs.microsoft.com/en-us/partner-center/develop/get-agreement-metadata](https://docs.microsoft.com/en-us/partner-center/develop/get-agreement-metadata)

-   [https://docs.microsoft.com/en-us/partner-center/develop/confirm-customer-consent](https://docs.microsoft.com/en-us/partner-center/develop/confirm-customer-consent)

**E se eu estou tendo dificuldades para implementar a API do Partner Center?**

Aceitação poderá ser confirmada manualmente por meio do painel do Partner Center.

**NOVOS | Como uma alternativa para usar o painel do Partner Center ou a API do Partner Center, posso em massa carregar um arquivo com as informações de confirmação do cliente em vez disso?**

Neste momento, o painel do Partner Center e a API do Partner Center são os únicos meios aceitos fornecer confirmação de aceitação do cliente do MCA.

**NOVOS | Sou um parceiro usando uma plataforma de terceiros, ou "middleware", que executa a funcionalidade para conectar sistemas da minha empresa com o Partner Center. O que preciso fazer?**

Os parceiros usando plataformas que se integram a API do Partner Center aconselhável Fale diretamente com os provedores em relação a seus planos e cronogramas para integrar a funcionalidade de API do Partner Center que automatiza o processo de confirmação do cliente.

**NOVOS | Você pode fornecer um exemplo de fluxo de API para importar o nome completo do cliente, o endereço de email e a data de aceitação MCA?**

Os parceiros devem examinar a documentação de API do Partner Center para obter detalhes e exemplos.

**NOVOS | Como obter o MCA mais recente para exibir para o cliente e usar o registro de confirmação como parte do processo minha API?**

API do Partner Center atual é muito básico e não fornece para parceiros recuperar programaticamente uma cópia do MCA, nem ele capturar versões do MCA aceita por um cliente como parte de confirmação. 

**NOVOS | Usando a API do Partner Center para esse processo requer um parâmetro userID. Como gerenciar essa exigência quando são colocados pedidos por meio de plataformas de terceiros que não se integram com o Azure AD?**

Nesse caso, a ID de usuário identifica o funcionário sob o locatário parceiro que está fornecendo a confirmação de consentimento do cliente. Se o funcionário não tem uma conta de usuário do Azure AD correspondente sob o locatário do parceiro, é aconselhável designar uma conta de usuário específicas para essa finalidade e criar um processo para capturar separadamente a identidade do funcionário fornecendo o confirmação. 

**NOVOS | Por que a Microsoft está exigindo o parceiro confirmar o consentimento do cliente para o MCA em vez de obtenção diretamente do cliente?**

O cliente estiver comprando do parceiro CSP, não diretamente da Microsoft. Para usar os serviços que a Microsoft fornece, a Microsoft fornece termos que estão documentados no MCA. O MCRA exige que os parceiros CSP tiver clientes Examine e aceite o MCA antes de fazer um pedido. 

**NOVOS | Estou usando API e o painel do Partner Center interface do usuário da web para confirmar a aceitação do cliente MCA. Depois de 7 de novembro de 2018, quais são as opções para confirmar a aceitação do cliente?**

As duas opções estão disponíveis e aceitável para confirmar se a aceitação do cliente MCA. Transações usando o painel do Partner Center exigirá a confirmação de estar presente a partir de 7 de novembro de 2018. Transações usando a API do Partner Center exigirá a confirmação de estar presente começando 22 de março de 2019.
